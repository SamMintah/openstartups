# Replication Checklist

Use this checklist when creating a replication guide for a new startup.

## Required Sections

Every replication guide MUST include:

### Overview
- [ ] One paragraph: what it does, who it's for, why it works
- [ ] Key metrics (MRR, rank, growth, team size)

### Market Opportunity
- [ ] Specific niche and why it's underserved
- [ ] Competitive landscape (who else plays here)
- [ ] Why now (timing factors)

### MVP Build Guide
- [ ] Core features list (must-have for v1, with implementation detail)
- [ ] Tech stack table (Layer | Tool | Cost | Why)
- [ ] No-code alternative path
- [ ] Estimated build time and cost

### Pricing Strategy
- [ ] Current tiers (if known)
- [ ] Pricing psychology and rationale
- [ ] Positioning against competitors

### Customer Acquisition
- [ ] Primary channels (specific tactics, not generic)
- [ ] Content/SEO strategy
- [ ] Early traction tactics (first 10, first 100 customers)

### Growth Levers
- [ ] Viral loop or referral mechanic
- [ ] Partnership opportunities
- [ ] Expansion revenue paths

### Unit Economics (estimates)
- [ ] ARPU, estimated CAC, payback period, LTV

### Common Pitfalls
- [ ] Top 3-5 mistakes with avoidance strategies

### Tools & Resources
- [ ] Specific tools with links
- [ ] Communities to join

### Launch Checklist
- [ ] Pre-launch (Week 1-2) items
- [ ] Launch (Week 3) items
- [ ] Post-launch (Week 4+) items

## Quality Standards

- Zero placeholder text — every section must contain unique, specific content
- Include real numbers, tool names, and tactical details
- Write as if advising someone who is about to build this tomorrow
- Attribution footer at bottom
